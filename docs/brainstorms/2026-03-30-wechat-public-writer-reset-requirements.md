# WeChat Public Writer Reset

## Problem Frame

The workflow now succeeds operationally: it can discover a hot topic, build evidence, generate a draft package, and push a real WeChat draft after human approval.

But the public article layer is failing the actual publishing job. The system often exposes internal research scaffolding directly to readers, including operator phrases, verification structure, source-brand leakage, and editing instructions that belong to the editor rather than the audience.

The result is a draft that is technically complete but editorially unusable. The user still needs to rewrite too much by hand before publication.

## Requirements

**Public Writing Separation**
- R1. The system must keep the evidence brief, verification scaffolding, and risk notes as internal artifacts rather than rendering them into the default reader-facing article.
- R2. The default article must be generated by a dedicated public-writer layer that turns evidence inputs into readable公众号文章 prose.
- R3. The public-writer layer must support framework-based writing so the article structure matches the topic type instead of reusing a single analyst-style scaffold.

**Reader-Facing Copy Quality**
- R4. Default public titles must not include source outlet branding, fact-check phrasing, or operator language such as “哪些已经确认，哪些仍未确认”.
- R5. Default subtitles and opening paragraphs must read as article prose, not internal workflow narration.
- R6. Default sections must explain why the topic matters, what changed, and what to watch next in reader-facing language.
- R7. Source references must remain available, but the default presentation should favor an end-of-article source appendix rather than inline body clutter.

**Topic Desk Controls**
- R8. The hot-topic stage must support operator control over topic preference, exclusion rules, score weights, and minimum thresholds.
- R9. The discovery result must make those active controls visible so the operator can understand why a topic won.

**Review And Editing**
- R10. Human review must remain mandatory before any real WeChat push.
- R11. Editor anchors may still exist, but the default publishable HTML must not expose them inline unless the operator explicitly chooses that mode.
- R12. The package must continue to preserve enough review metadata for a human editor to quickly personalize the draft before pushing.

**Cover Policy**
- R13. The cover workflow must prefer dedicated no-text or near-no-text cover candidates ahead of body-image fallback.
- R14. The cover strategy must stay compatible with the existing WeChat draft payload contract and future real push steps.

**Quality Guardrails**
- R15. The workflow must catch obvious operator-language regressions before packaging a draft as publish-ready.
- R16. The improved writing layer must not weaken the existing evidence firewall or human review gate.

## Success Criteria

- A default generated draft reads like a real公众号文章初稿 rather than an internal evidence memo.
- Titles no longer leak source brands or fact-check scaffolding.
- The operator can tune hot-topic ranking without code edits.
- Sources remain traceable, but the article body is no longer overloaded with internal citations and workflow notes.
- The package still works with the existing WeChat draft push flow.

## Scope Boundaries

- Do not remove the evidence-first architecture.
- Do not remove the human review gate before WeChat draft push.
- Do not solve this by pretending weak evidence is strong evidence.
- Do not expand this batch into multi-platform publishing.

## Key Decisions

- Internal brief and public article are separate products: the first is for safe reasoning, the second is for readers.
- Human review stays in the loop by design, not as a fallback.
- Topic-desk control is part of product usability, not an optional advanced setting.
- Cover images should default toward text-free visual safety for WeChat use.

## Dependencies / Assumptions

- The current evidence bundle remains the source of truth for confirmed facts, unresolved questions, and source references.
- The existing WeChat packaging and push contract stays stable while the writing layer is replaced.
