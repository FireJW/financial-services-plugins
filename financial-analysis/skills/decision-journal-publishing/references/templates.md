# Templates

## Free WeChat Post Template

Use this structure:

1. `一句话结论`
2. `已确认`
3. `未确认 / 仅推断`
4. `最关键的冲突点`
5. `接下来要盯的 3 个变量`
6. `什么会改变我的判断`
7. CTA: `领取 Lite 模板` or `进入模板包等候名单`
8. mandatory footer

Packaging rule:

1. if the post is top-of-funnel, lead the title with a real event, affected
   group, or concrete consequence
2. put the method term inside the body or subtitle, not always in the headline

## Mandatory Footer

Every public finance post should end with:

1. `分析时点：`
2. `适用场景：事件阅读 / 研判训练 / 模板示范`
3. `内容用途：仅作研究与信息整理，不构成个性化投资建议`
4. `失效方式：如关键触发条件未发生，判断不升级`

## Toutiao Fast Card Template

1. title with one sharp question or one concrete implication
2. one-line takeaway
3. confirmed vs unconfirmed
4. one conflict point
5. one next watch item
6. CTA to WeChat
7. one-line boundary statement if the topic is sensitive

Headline rule:

1. prefer `who is affected + what changed + why it matters`
2. avoid headlines that start with internal method jargon such as `冲突矩阵`,
   `模板骨架`, or `证据分层`

## Xiaohongshu Slice Template

Best formats:

4. "3 mistakes" explainer

Suggested structure:

1. scenario headline
2. three-step template fragment
3. one warning
4. one save-worthy visual
5. CTA to get the full pack

## Weekly Review Template

1. this week's top call
2. what played out
3. what invalidated prior assumptions
4. what stays on the watchboard
5. scorekeeping note
6. next week's focus

## Public Scoreboard Template

Use this instead of "I predicted it" claims.

Columns:

1. date published
2. original scenario
3. trigger condition
4. invalidation condition
5. outcome status
6. lesson

## Lite Template Lead Magnet

Include:

1. decision template skeleton
2. evidence classification legend
3. one example screenshot
4. one CTA to the paid pack
5. one short note for who this is for

Do not include:

1. full scoring logic
2. full sample library
3. workflow details
4. language that implies guaranteed outcomes

## Paid Template Pack Outline

1. who this pack is for
2. when to use it
3. main template
4. scoring or weighting logic
5. trigger tree
6. conflict matrix
7. AI workflow map
8. three sample cases
9. update log
10. compliance block

## Compliance Block For Paid Pages

Every paid-product page should include:

1. `这不是荐股或收益承诺产品`
2. `适合谁 / 不适合谁`
3. `案例仅用于展示模板用法，不构成个股建议`
4. `模板提供的是阅读结构，不替代独立判断`

## Safe Forward-View Language

Preferred phrases:

1. `当前更高概率的路径是`
2. `如果 A 发生，判断升级为`
3. `如果 B 发生，原判断失效`
4. `现阶段更像是`
5. `这不是结论，是当前最优读法`
6. `以下判断基于 {date/time} 之前的公开信息`

Avoid:

1. `我敢打包票`
2. `这次一定`
3. `精准预言`
4. `闭眼上`
5. `稳赚`

## Benchmark-Aligned Outer Packaging

High-read benchmark cases on WeChat and Toutiao usually package serious content
with one of these outer shells:

1. a public event that already has attention
2. an affected group under pressure
3. one concrete puzzle the reader wants resolved
4. one obvious business or market consequence

Your inner structure can stay evidence-first and method-heavy.

Your outer packaging should still feel public-facing.
