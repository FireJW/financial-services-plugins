# Operating Playbook

## Weekly Cadence

Default cadence for a solo operator:

1. WeChat: 2 anchor posts per week
2. Toutiao: 3 fast cards per week
3. Xiaohongshu: 0-1 reuse slice per week, optional
4. X: none in the first 30 days

This is enough to test demand without turning the business into a content treadmill.

## Weekly Rhythm

### Monday

1. use `news-index` or `article-auto-queue` to rank candidate topics
2. choose one lead topic and one backup topic
3. update the topic board and CTA focus for the week

### Tuesday

1. build the evidence pack
2. publish one Toutiao fast card
3. draft the first WeChat anchor

### Wednesday

1. publish the first WeChat anchor
2. collect comments, saves, and lead-magnet feedback
3. patch the sales copy if friction is obvious

### Thursday

1. publish the second Toutiao fast card
2. refine the template pack or lite template
3. reuse to Xiaohongshu only if the material is naturally save-worthy

### Friday

1. publish the second WeChat anchor or weekly review
2. publish the third Toutiao fast card
3. update the public review log

### Weekend

1. rest or batch sample-library work
2. review buyer feedback
3. prepare the next week's queue

## Production System

The default production loop is:

1. select one high-relevance event or recurring event type
2. build one evidence pack
3. publish one WeChat long-form anchor
4. cut one to three acquisition wrappers from the same pack
5. push one CTA only
6. log performance and buyer feedback

The real unit of work is the evidence pack, not the number of posts.

## Benchmark Index Routine

Keep a rolling benchmark index beside the content plan.

Minimum weekly review:

1. two WeChat cases
2. two Toutiao cases
3. one scheduled `benchmark-refresh` run that updates the observation log and candidate inbox

When possible, include at least:

1. one `10W+ acquisition` case
2. one `commercial-fit` case
3. one `mixed` case that has both reach and monetization clues

Track these fields:

1. platform
2. URL
3. date
4. read band
5. account type
6. topic type
7. hook
8. affected group
9. decision density
10. commercial fit
11. CTA
12. what to copy
13. what to avoid

Do not review reference cases as inspiration only. Review them as structured
records.

Daily maintenance rule:

1. refresh into the observation log every 24 hours
2. keep the curated library stable unless a human promotes or archives a case
3. score only `reviewed` cases by default

## 30-Topic Structure

Use six repeating pillars:

1. evidence discipline
2. scenario and trigger logic
3. template anatomy
4. worked cases
5. review and correction
6. operator perspective and information diet

Bring AI in only after the audience already understands the judgment framework.

At least half of the traffic-facing posts should start from a real event, affected
group, or public consequence. Do not fill the top of funnel with internal method
jargon.

## 90-Day Rollout

### Days 1-30

1. lock one narrow event type or one repeatable event-reading frame
2. launch one lead magnet
3. launch one paid product only: template pack v1
4. publish repeated-structure content on WeChat and Toutiao
5. complete one real payment loop

### Days 31-60

1. improve the same pack into v1.1 if buyer usage is clear
2. tighten the sales page using buyer objections
3. expand the sample library
4. decide whether Xiaohongshu reuse is worth the extra effort
5. do not launch membership unless users explicitly want update rights

### Days 61-90

1. decide whether the core pack deserves pack #2 or membership
2. build a template library index
3. formalize the weekly review archive
4. test one adjacent channel or partnership only after the core loop is stable

## Workflow Integration

Map repo commands to the publishing system:

1. `news-index` for evidence-first current-state notes
2. `news-refresh` when the same story needs only recent-window updates
3. `article-workflow` for one long-form package plus revision template
4. `article-auto-queue` when candidate-topic volume rises
5. `article-revise` for platform-specific or tone-specific second passes

## Compliance Operating Rules

Every public finance post must include:

1. analysis time or time window
2. confirmed vs unconfirmed separation
3. usage boundary
4. invalidation or "what would change my view"

Every weekly cycle must also include:

1. one public review log
2. one correction when a prior read weakens or fails
3. no certainty language or target-price claims

## Decision Rules

If time compresses:

1. cut Xiaohongshu first
2. reduce Toutiao second
3. keep WeChat anchors
4. keep the lead magnet and pack maintenance

If Toutiao grows much faster than WeChat:

1. use it for more acquisition
2. do not let it redefine the tone
3. move stronger readers upward into WeChat

If comments skew toward "which stock should I buy":

1. tighten the boundary language
2. increase event-reading and invalidation education
3. do not redesign the product around retail tip demand
