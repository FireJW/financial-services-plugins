# Feedback Workflow Command

## Dual-Track Hints

- route through `routing-index.md` before broad browsing
- treat fresh-source collection or moving-fact work as `Complex`
- use `financial-services-docs/docs/runtime/codex-dual-track/complexity-classification.md` to decide escalation
- finish with the delivery contract and explicitly state where human judgment still enters the workflow

Use this command when the user wants one runnable flow that:

1. collects interviews, talks, podcasts, notes, or raw feedback inputs
2. builds a source-traceable evidence board
3. separates direct quote, summary, and inference
4. reconstructs or operates a feedback-to-iteration workflow
5. emits a practical artifact such as a priorities brief, SOP, or weekly loop

Good fits:

- "collect what this leader said about using AI with user feedback"
- "turn these interviews into an operating workflow"
- "summarize support + calls + social posts into a weekly design priorities loop"
- "design a research-preview feedback cadence"

Native skill:

- `financial-analysis/skills/feedback-iteration-workflow/SKILL.md`

Default output:

- one-line thesis
- source board
- attributable claim inventory
- workflow table
- cadence recommendation
- risks / overclaim guardrails
